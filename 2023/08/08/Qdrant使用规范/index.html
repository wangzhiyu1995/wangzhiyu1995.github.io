Qdrant开发操作规范

集合命名规范

  1. 命名组成一般为：集合名+版本id，用下划线连接，再起别名用于业务读写
  2. 命名使用具有意义的英文词汇，词汇中间以下划线分隔
  3. 命名只能使用英文字母、数字、下划线，以英文字母开头
  4. 命名使用小写字母

示例：
你想创建一个名为chat_cm的集合，则创建时集合名设定为chat_cm_1，创建完成后再为chat_cm_1这个集合创建别名chat_cm用于业务读写。
这样做的好处是后续更新了chat_cm_2版本的向量，则可以使用别名切换至新的集合，不需要调整代码中配置的集合名。

创建集合示例

创建集合时需要指明：

  1. size 向量的维度
  2. distance 距离度量的类型：有dot/cosine/Euclidean三种
  3. shard_number 集合分片数：目前有3个节点，按照官方建议6个分片
  4. replication_factor 集合副本数：目前有3个节点，按照官方建议每个节点2个副本

更喜欢低内存占用和高速搜索

实现低内存占用的高速搜索的主要方法是将向量持久化在磁盘上，同时最小化磁盘读取的数量。

延迟和吞吐量

有两种方法来衡量搜索速度：

  • 请求延迟：从提交请求到收到响应的时间
  • 吞吐量：系统每秒可以处理的请求数

这些方法并不互斥，但在某些业务场景，最好对其中一种方法进行优化。

为了最大限度地减少延迟，您可以设置qdrant为单个请求使用尽可能多的cpu核数。 通过将集合中的segment数设置为等于系统中的cpu核数来实现这一点。在这种情况下，每个segment将被并行处理，且将更快地获得最终结果。

为了获得更好的吞吐量，您可以为qdrant设置尽可能多的cpu核数来并行处理多个请求。 为此，您可以配置qdrant以使用最小数量的segment，通常为2。 较大的segment的好处是索引的大小和找到最近邻所需的比较次数更少。但同时需要更多的时间来建立索引。

直接上传到磁盘

当你上传的向量不能全部放入RAM时，你可能希望使用memmap支持。

在创建集合时，可以使用on_disk参数对每个向量启用memmap。该配置始终将向量数据直接存储在磁盘上，适用于传入大量数据的场景。

在这种情况下，不建议使用memmap_threshold_kb。它需要优化器不断地将内存中的segment转换为磁盘上的memmap segment。这个过程比较慢，而且在采集大量数据时，优化器可能会成为瓶颈。

更多信息请参阅配置Memmap存储。

并行上传到多个分片

在Qdrant中，每个集合都被拆分为分片。每个分片都有一个单独的预写日志（WAL），负责排序操作。 通过创建多个分片，您可以并行上传大型数据集。每台机器2到4个分片是合理的数量。

以存储为中心的配置

如果您的业务优先级是以平均搜索延迟处理大量向量，建议配置mmap存储。 在这种情况下，向量将存储在磁盘上的内存映射文件中，只有最常用的矢量将保存在RAM中。

可用RAM的数量将直接影响搜索的性能。 根据官方经验，如果在RAM中保留的向量少2倍，则搜索延迟将降低2倍。

当训练模型时，如果向量的维度设置过大，会导致几个问题：

1.内存占用：更高的维度意味着需要更多的内存来存储模型参数和计算中间结果。如果维度设置过大，可能会超出可用的内存限制，导致训练无法进行或者效率低下。

2.过拟合：过大的向量维度可能会导致过拟合问题。过拟合是指模型在训练数据上表现良好，但在未见过的测试数据上表现较差。高维度的向量可以容易地记忆训练数据的细节，而无法泛化到新的数据。

3.计算复杂度：更高的维度意味着需要进行更多的计算操作。这会增加模型训练和推断的时间成本。如果维度设置过大，训练和推断的速度可能会变得非常缓慢，不太适用于实时应用或大规模数据集。

因此，在选择向量的维度时，需要在资源限制、模型性能和计算效率之间进行权衡。通常情况下，适当的维度可以在保持模型性能的同时，提供足够的计算效率和内存使用。可以通过实验和验证来确定最佳的维度设置。


词向量的维度设置需要考虑多个因素，包括语料库的规模、任务的复杂度和可用的计算资源。一般来说，较大的维度可以捕捉更多的语义信息，但也会增加计算成本。较小的维度可能无法充分表示词语之间的关系。通常，词向量维度的范围在100到300之间。

对于小规模的数据集或简单任务，可以选择较小的维度，如100维。这样可以节省计算资源，并且可能足够表达基本的语义关系。对于大规模数据集或复杂任务，可以选择更大的维度，如200维或300维。这样可以更准确地捕捉更丰富的语义信息。

在实践中，可以尝试不同的维度设置，并通过与任务的性能进行评估来确定最佳维度。还可以考虑使用预训练的词向量模型，如Word2Vec、GloVe或BERT，它们通常具有经过优化的维度设置。


确定词向量的维度大小是使用嵌入(embedding)训练词向量时的一个重要决策。以下是一些常见的方法来确定词向量的维度大小：

1. 基于任务的性能：一种常用的方法是根据具体任务的性能来选择合适的维度大小。训练不同维度大小的词向量，并在特定任务上评估它们的性能，选择在该任务上表现最佳的维度大小。

2. 经验值：根据经验，一般来说，较小的维度（例如50或100）可以捕捉到一些基本的语义信息，而较大的维度（例如200或300）可能能够更好地捕捉更丰富的语义信息。可以根据应用场景和数据规模选择一个合适的经验值。

3. 数据规模：数据规模也会影响到选择词向量维度大小。通常来说，较大的数据集可以支持更高维度的词向量，因为它们可以提供更多的样本来训练更复杂的模型。相反，如果数据集较小，选择较小的维度可能更合适，以避免过拟合。

4. 先验知识：某些特定领域或任务可能有先验知识可用，可以根据这些知识来选择适当的维度大小。例如，某些任务可能需要特定的维度大小来对应特定的语言特征。

总的来说，选择词向量的维度大小需要综合考虑任务性能、经验值、数据规模和先验知识等因素。在实践中，可以尝试不同的维度大小，并使用评估指标来选择最佳的维度大小。
