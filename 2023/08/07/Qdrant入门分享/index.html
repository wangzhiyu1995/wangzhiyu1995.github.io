什么是向量数据库

什么是向量

在数学中，向量是有大小和方向的量，可以使用带箭头的线段表示，箭头指向即为向量的方向，线段的长度表示向量的大小。两个向量的距离或者相似性可以通过点积、欧式距离或者余弦距离得到。
向量是对象或数据点的数学表示，其中向量的每个元素对应了对象的特定特征或属性。例如，在图像识别系统中，向量可以表示图像，向量的每个元素表示像素值或该像素的描述符/特征。在音乐推荐系统中，每个向量表示一首歌，向量的每个元素表示一首歌曲的特征，如节奏、类型、歌词等。

向量数据库介绍

向量嵌入技术

向量嵌入（vector embedding）是一种将非数值的词语或符号编码成数值向量的技术。它是自然语言处理（NLP）和深度学习中常用的预处理技术。
通常，向量嵌入是通过一个神经网络来学习的，该网络接收文本中的词语作为输入，并输出一个对应的词向量，其中词向量是一个数值向量，每个数值代表词语的某个特征。例如，通过向量嵌入，我们可以将词语”dog”和”cat”表示为两个不同的数值向量，并可以通过计算两个向量的距离来判断它们的相似度。

什么是向量数据库

向量数据库是一种用于高效存储和查询高维向量的数据库。
在传统的OLTP和OLAP数据库中(如下图所示)，数据被组织成行和列(这些被称为表)，查询是基于这些列中的值执行的。而在某些应用程序中，如图像识别、自然语言处理和推荐系统等，数据通常表示为高维空间中的向量，这些向量加上id和有效载荷，就被称为存储在像qdrant这样的向量数据库的集合(Collection)中的元素。
简言之，向量数据库就是用来存储，检索，分析向量的数据库。只所以称之为数据库，是因为它有下面几个特征：

  • 提供标准的sql访问接口，降低用户的使用门槛。
  • 提供高效的数据组织，检索和分析的能力。一般用户在存储和检索向量的同时，还需要管理结构化的数据，即支持传统数据库对结构化数据的管理能力。

向量数据的应用

语义搜索

文本搜索通常有两种方式：词法搜索和语义搜索。词法搜索是基于模式寻找精确单词或字符串匹配，就是平时我们常说的关键字匹配。语义搜索则将搜索查询或问题的含义放入上下文中，以理解文本的含义和上下文，并获得更准确和相关的搜索结果。 向量数据库，存储和索引自然语言处理模型中的向量嵌入，以更好地理解文本字符串、句子和整个文档。使用语义搜索可以让用户更快地找到所需内容，无需了解数据分类。这不仅提供了更好的用户体验，还能提高效率。

对非结构化数据进行相似度搜索

传统数据库难以分类和存储非结构化数据集，如图像、音频、视频等。对每个对象手动应用关键字、描述和元数据也很繁琐。不同人对复杂数据对象的分类可能有所不同，使得搜索变得随意。向量数据库能够更好理解数据，对数据进行相似度搜索。

推荐排序

向量数据库可用于驱动排名和推荐引擎。向量数据库具备寻找相似物品的能力，因此它成为提供相关建议和轻松排名物品的理想选择。相似度分数也可用于对物品进行排序。因此电商领域，可以用它为用户提供与过去购买或当前正在研究的物品相似的建议。流媒体服务（音乐、短视频）可以根据用户的歌曲评级创建个性化推荐。

什么是qdrant

qdrant介绍

qdrant是一个向量相似性搜索引擎（即向量数据库），它提供了一个可用于生产环境的服务，具有方便的API来存储、搜索和管理点（即向量），并具有额外的负载。你可以将有效载荷视为额外的信息，可以帮助你专注于搜索，也可以接收到可以提供给用户的有用信息。

qdrant相关概念

架构图

上图显示了qdrant的一些主要组件。我们需要熟悉这些组件及术语：

  • 集合(collection): 集合是一组命名的点(带有有效载荷的向量)，你可以在其中搜索。同一个集合中的向量可以有不同的维度，可以用唯一的度量进行比较。

  • 距离度量(distance): 用于度量向量之间的相似性，必须在创建集合的同时指定。度量的选择取决于向量的获取方式，特别是用于编码新查询的神经网络。

  • 点(Points): Points是qdrant操作的核心实体，它们由向量、可选的id和可选的有效载荷组成。

  • Id: 向量的唯一标识符。

  • 向量(Vector): 数据的高维表示，例如图像、声音、文档、视频等。

  • 有效载荷(payload): 有效载荷是一个JSON对象，可以向vector中添加额外的数据。

  • 存储(storage): qdrant可以使用两种存储方式中的一种，内存存储(将所有向量存储在RAM中，速度最快，因为只有在持久化时才需要访问磁盘)或Memmap存储(在磁盘上创建与文件关联的虚拟地址空间)。

  • 客户端(client): 您可以用来连接到qdrant的编程语言。

集合的概述

集合(collection)是一组有名称的点（带有有效载荷的向量），你可以在其中进行搜索。同一个集合中的向量必须具有相同的维度，并且只能用一个距离度量进行比较。
距离度量用于度量向量之间的相似性。度量的选择取决于向量获取的方式，尤其是神经网络编码器训练的方法。
qdrant支持以下最流行的指标类型：

  • Dot product 点积
  • Cosine similarity 余弦相似性
  • Euclidean distance 欧式距离

集合别名

在生产环境中，有时需要无缝切换不同版本的向量。例如，当升级到新版本的神经网络时。
在这些情况下，停止服务并使用新向量重新生成集合显然是不合适的。别名是现有集合的附加名称。对集合的所有查询也可以使用别名，使用别名和使用集合名称查询会返回相同的结果。

有效荷载 payload

概述

在向量数据库中，”payload”是指与每个向量相关联的附加数据或信息。简单来说，它是存储在数据库中的向量的额外内容。

想象一下，你有一个向量数据库，其中存储了一些图片的特征向量。除了这些向量之外，你可能还想存储图片的文件名、拍摄日期、标签等其他信息。这些额外的信息就是payload。

使用payload可以使数据库更加灵活和丰富。当你查询数据库时，可以同时获取向量和与之相关联的其他数据。例如，在搜索相似图片时，你可以获取相似向量，并通过payload获取图片的文件名或标签等信息。

总之，payload是向量数据库中与每个向量关联的附加数据，使得数据库查询更为方便和有用。

Qdrant允许您存储任何可以使用JSON表示的信息。

搜索

相似性搜索

搜索最近的向量是许多表征学习应用程序的核心。现代神经网络被训练成将对象转换为向量，使得真实的世界中接近的对象在向量空间中看起来接近。例如，它可以是具有相似含义的文本、视觉上相似的图片或相同流派的歌曲。

向量数据库是为高效存储和查询这些高维向量而优化的，通常使用专门的数据结构和索引技术，如ANN搜索领域基于图的算法：近似的最近邻搜索(HNSW)。这些数据库支持快速的相似性和语义搜索，同时允许用户根据某种距离度量找到与给定查询向量最近的向量。最常用的距离度量是欧氏距离、余弦相似度和点积，这三种距离度量在Qdrant中都得到了充分的支持。
计算向量之间相似度的方法有很多种。在Qdrant术语中，这些方法称为指标(metrics)。度量的选择取决于向量的获取，特别是神经网络编码器训练的方法。
Qdrant支持以下几种最流行的指标类型:

  • 余弦相似度：余弦相似度(Cosine Similarity)是一种度量两个事物相似程度的方法。把它想象成一个尺子，它告诉你两个点之间的距离有多远，但它不是测量距离，而是测量两个东西的相似程度。它经常用于文本，用于比较两个文档或句子的相似程度。余弦相似度的输出范围是0到1，其中0表示这两个东西完全不相似，1表示这两个东西完全相同。这是比较两个东西的一种直接而有效的方法!
  • 点积：
    点积相似度(dot product similarity)是另一种度量两个事物相似程度的方法，类似于余弦相似度(cosine similarity)。它经常用于机器学习和数据科学中处理数字。点积相似度的计算方法是将两组数字相乘，然后相加。和越大，表示两组数字越相似。它就像一个刻度，告诉你两组数字之间的匹配程度。
  • 欧几里得距离:
    欧几里得距离(Euclidean Distance)是一种测量空间中两点之间距离的方法，类似于我们如何测量地图上两个地方之间的距离。计算方法是求两个点坐标差值平方和的平方根。在机器学习中，这种距离度量通常用于衡量两个数据点的相似程度或不相似程度。

查找规划策略

根据搜索中使用的过滤器，查询执行有几种可能的场景。Qdrant根据可用的索引、条件的复杂性和过滤结果的基数选择一个查询执行选项。这个过程称为查询规划。
策略选择过程很大程度上依赖于启发式方法，并且在不同版本中可能有所不同。但是，一般原则是:

  • 对每个段独立执行规划(有关段的更多信息，请参见存储)
  • 如果点的数量低于阈值，首选全扫描
  • 在选择策略之前，估计过滤结果的基数
  • 如果基数低于阈值，则使用有效负载索引(请参见索引)检索点
  • 如果基数高于阈值，则使用可筛选的向量索引

可以使用配置文件或独立地为每个集合调整阈值。
例如：寻找类似于向量[0.2, 0.1, 0.9, 0.7]的向量。参数limit（或其别名- top）指定我们希望检索的最相似结果的数量。
配置中params键下的值可以指定搜索的自定义参数：

  • hnsw_ef 指定HNSW算法的ef参数的值。
  • exact 不使用近似搜索（ANN）的选项。如果设置为true，搜索可能会运行很长时间，因为它会执行完整扫描以检索准确的结果。

按分数过滤结果

除了有效负载过滤之外，过滤掉具有低相似性分数的结果可能是有用的。例如，如果您知道模型的最小接受分数，并且不希望任何结果低于阈值。在这种情况下，您可以使用搜索查询的score_threshold参数。它将排除所有得分低于给定值的结果。
参数值取决于所使用的度量，该参数可以排除较低或较高的分数。例如，欧几里德度量的较高分数被认为是更远的，并且因此将被排除。

过滤条件

payload中不同类型的值对应于我们可以应用于它们的不同类型的查询。让我们看看现有的条件变量以及它们适用于哪些类型的数据。

Match

最简单的条件是检查存储的值是否等于给定的值。如果存储了多个值，则其中至少有一个值应与条件匹配。您可以将其应用于关键字，整数和bool有效载荷。

Match any

如果要检查存储的值是否为多个值之一，可以使用“匹配任何值”条件。Match Any用作给定值的逻辑OR。它也可以被描述为IN操作符。
可以将其应用于关键字和整数有效载荷。

Match except

如果要检查存储的值是否不是多个值中的一个，可以使用Match Except条件。Match Except用作给定值的逻辑NOR。它也可以被描述为NOT IN操作符。

配置加载顺序和优先级

启动时可能会加载多个配置。所有这些都合并到一个Qdrant使用的单一有效配置。
配置按以下顺序加载：

  1. Embedded base configuration (source)
  2. File config/config.yaml
  3. File config/{RUN_MODE}. yaml (such as config/production.yaml)
  4. File config/local.yaml
  5. Config provided with –config-path PATH (if set)
  6. Environment variables

此列表从最不重要到最重要。以后配置中的属性将覆盖之前加载的内容。启动时用–config-path指定的配置将覆盖1-4文件中的那些配置。

这些文件中的大多数默认包含在Docker容器中。但如果您用二进制方式部署qdrant，则本地服务器上可能没有这些配置文件，需要手动添加。
如果找不到文件2或3，启动qdrant时会显示警告。如果提供了文件5但未找到，则在启动时显示错误。

存储

集合存储

一个集合中的所有数据都被划分为段。每个段具有其独立的向量和有效载荷存储以及索引。

存储在段中的数据通常不重叠。然而，将相同的点存储在不同的段中不会引起问题，因为搜索包含去重机制。这些段由向量和有效负载存储、向量和有效负载索引以及id映射器组成，id映射器存储内部id和外部id之间的关系。集合中的段的配置可以不同并且彼此独立，但是集合中必须至少存在一个“可追加”段。

向量存储

根据应用程序的要求，Qdrant可以使用其中一种数据存储选项。必须在搜索速度和使用的RAM大小之间做出选择。

  • RAM存储：将所有向量存储在RAM中，具有最高的速度，因为磁盘访问仅用于持久性。
  • Memmap存储：创建与磁盘上的文件相关联的虚拟地址空间。内存映射文件不会直接加载到RAM中。相反，它们使用页面缓存来访问文件的内容。该方案允许灵活使用可用存储器。有了足够的RAM，它几乎与RAM存储一样快。 上面的操作将创建一个集合，其中所有向量立即存储在memmap存储中。如果您的Qdrant实例使用快速磁盘运行，并且您正在处理大型集合，则建议使用此方法。
    设置memmap_threshold_kb选项。此选项将设置阈值，超过该阈值后，段将被转换为memmap存储。

有两种方法可以做到这一点：

  • 在配置文件中全局设置阈值。参数名为memmap_threshold_kb。

  • 分别为每个收集设置阈值 post 或 put

设置memmap阈值参数的经验法则很简单：

  • 如果你的业务处于较均衡的使用场景-设置memmap阈值与indexing_threshold相同（默认值为20000）
  • 如果你的业务处于高写负载和低RAM -设置memmap阈值低于indexing_threshold，如10000

有效荷载存储

Qdrant支持两种类型的有效负载存储：内存和磁盘。

  • InMemory 有效负载存储的组织方式与内存中向量相同。有效负载数据在服务启动时加载到RAM中，而磁盘和RocksDB仅用于持久化。这种类型的存储工作速度相当快，但它可能需要大量的空间来将所有数据保存在RAM中，特别是如果有效载荷附加了大量的值-文本甚至图像的摘要。
  • OnDisk 在负载值较大的情况下，最好使用OnDisk负载存储。这种类型的存储将直接读取和写入有效负载到RocksDB，因此它不需要任何大量的RAM来存储。然而，缺点是访问延迟。如果您需要查询具有某些基于有效负载的条件的向量-检查存储在磁盘上的值可能会花费太多时间。在这种情况下，我们建议为过滤条件中使用的每个字段创建有效负载索引，以避免磁盘访问。创建字段索引后，Qdrant将在RAM中保留索引字段的所有值，而不管有效负载存储类型如何。

您可以在创建集合期间使用配置文件或集合参数on_disk_payload指定所需的有效负载存储类型。

版本控制

为确保数据完整性，Qdrant分2个阶段执行所有数据更改。在第一步中，将数据写入预写日志（WAL），该日志对所有操作进行排序并为其分配一个序列号。

一旦将变更添加到WAL，即使发生断电，变更也不会丢失。然后，更改将进入分段。每个段存储应用于它的更改的最后版本以及每个单独点的版本。如果新更改的序列号小于点的当前版本，则更新程序将忽略该更改。该机制允许Qdrant在异常关闭的情况下安全且有效地从WAL恢复存储。

索引

Qdrant的一个关键特性是向量和传统索引的有效组合。这是必不可少的，因为要使向量搜索有效地使用过滤器，只有向量索引是不够的。简单来说，向量索引加速了向量搜索，而有效负载索引加速了过滤。

段中的索引独立存在，但索引本身的参数是为整个集合配置的。

向量索引

向量索引是通过特定数学模型在向量上构建的数据结构。通过向量索引，可以高效地查询与目标向量相似的多个向量。

Qdrant目前只使用HNSW作为向量索引。选择HNSW是因为HNSW与允许Qdrant在搜索期间使用过滤器的修改兼容良好。其次，根据公共基准测试，它是最准确和最快的算法之一。

HNSW（Hierarchical Navigable Small World Graph）是一种基于图的索引算法。它根据一定的规则为图像建立一个多层导航结构。在这种结构中，上层更稀疏，节点之间的距离更远。较低的层更密集，节点之间的距离更近。搜索从最上层开始，在该层中找到最接近目标的节点，然后进入下一层开始另一次搜索。经过多次迭代，它可以快速接近目标位置。

为了提高性能，HNSW将图的每一层上的节点的最大度限制为m。此外，您可以使用ef_construct（构建索引时）或ef（搜索目标时）指定搜索范围。

管理

Qdrant提供了管理工具，可以在运行时修改qdrant实例的行为，而无需手动更改其配置。

锁定

锁定API使用户能够限制qdrant进程上的可能操作。需要注意：

  • 配置不是永久性的，因此在重新启动后需要再次锁定。

  • 锁定仅适用于单个节点。在分布式部署设置中，有必要在所有所需节点上调用锁。

write标志启用/禁用写锁定。如果写锁设置为true，则qdrant不允许创建新集合或向现有存储添加新数据。但在写锁定后并不会禁止删除操作或更新。此功能使管理员能够防止qdrant进程使用更多磁盘空间，同时允许用户搜索和删除不必要的数据。
